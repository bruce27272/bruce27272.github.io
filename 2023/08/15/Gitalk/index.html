Hexo Next樣板開啟 Gitalk 留言版

發表於 2023-08-15 更新於 2023-08-18 分類於 技術紀錄 ， Web 閱讀次數：

今天試著幫我的部落格新增留言板

看了一些資料

蠻多人都是搭配Disqus

但有些人反映會收不到訊息

所以我採用Gitalk

Gitalk 是一個基于 GitHub Issue所開發的留言版套件

首先建立 GitHub OAuth APP

由於要使用 Gitalk 必須有 GitHub OAuth

所以先開啟 GitHub之後到 Setting

接下來選擇 Developer settings

然後點 OAuth Apps

未建立過OAuh 點Register a new application

接下來依照欄位填寫即可

Application name - 應用程式名稱    
Homepage URL - 應用程式網域 → 填部落格網址    
Application description - 應用程式描述    
Authorization callback URL - 授權回應網址 → 填部落格網址

參考如下

申請完之後你就會獲得這兩組 ID (Client ID、Client Secret)

請先複製起來，稍後會使用到

接下來就開啟 Next 佈景主題的 _config.yml

然後找到「Gitalk」的地方

就可以看到相關設定

依照欄位填入即可

enable: true    
github_id:  # GitHub 作者帳號    
repo: hsiangfeng.github.io # Repo 名稱，通常就是網域    
client_id: xxxxxxxx # 剛申請的 Client ID    
client_secret: xxxxxxxx # 剛申請的 Client Secret    
admin_user: hsiangfeng # 管理者的帳號    
distraction_free_mode: true # 無干擾模式    
language: zh-TW ## 語系

接下來只需要部屬就可以看到留言版開啟了

此外第一次使用會需要啟用 OAuth

只需要到任一篇文章內點一下啟用即可

只要有人在你的 Gitalk 留言時 Gitalk 就會在你的 GitHub repo 上開啟一個 issues

此外可以增加設定增加一行 labels 來做一下分類方便管理

enable: true    
github_id:  # GitHub 作者帳號    
repo: hsiangfeng.github.io # Repo 名稱，通常就是網域    
client_id: xxxxxxxx # 剛申請的 Client ID    
client_secret: xxxxxxxx # 剛申請的 Client Secret    
admin_user: hsiangfeng # 管理者的帳號    
distraction_free_mode: true # 無干擾模式    
language: zh-TW ## 語系    
labels: Gitalk

最後Gitalk 留言版套件可以使用 MD 格式同時支援高亮的

只要使用高亮的語法可以產生不同的效果
